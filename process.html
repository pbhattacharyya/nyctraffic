Process

We combined two separate datasets: one on motor vehicle incidents and one on weather patterns for the same time period. We used the date information to link both rows together.

After combining the dataset, we inputted the data into weka and used the linear regression classifier to build an algorithm.

We adjusted the time of the accident by 3 hours in order to include late night accidents into the data for the day before. We chose the 3 hour adjustment factor based on the data, which showed a natural dip between the end of one day and the beginning of the next.

Finding and Exploring the Data

Here are the steps we followed:


  1. We search for data sets relevant to our question.

  2. We brainstormed ways to correlate weather and other data.

  3. We used NYC's open data website to find our data set.

  4. We found a data set on traffic collisions in new york city for the year 2017.

  5. We found the weather data for the same year.

  6. We aggregated both datasets together based on date.

  7. We put the data into Weka.

Using Machine Learning

For the first pass, we used a linear regression algorithm.

  • Initially, we calculated our algorithm using all the features in our dataset.
  • Our correlation co-efficient was 0.3511
  • We realized all features are not independent of each other, such as number of people injured, number of motorcyclists injured, and number of bicyclists injured.
  • Because the features had continuous values, the algorithm was not as precise as it could be.

For the second pass, we took our dataset back to excel, and “binned” each variable into three or four values. For each type of data point, we used a different approach.

  • To bucket the number of accidents, we used the distribution of values and bucketed according to number of values in each group.
  • To bucket temperature, we used another dataset of average temperatures for NYC in each month. The values that are between the minimum and maximum values for each month were labeled to be in the as the average temperature bucket. Values outside that range was labeled as low or high.
  • To bucket wind speed, we used values for Beaufort scale which is a wind classification that are available online which classifies wind speeds in wind’s ability to create damage to property.
  • The binary features such as fog, thunder and snowfall were used as buckets.
  • After binning data points, we used the naive bayes and J48 classifier. The algorithm still gave us a low correlation.

For the third pass, we went back to excel and re-bucketed the number of accidents. We used the second and third quartiles as the average number of accidents. Values lower than that would be in a low number of accidents bucket, and values higher than that would be in a high number of accidents bucket.

We again used the naive bayes and J48 classifier. This time, due to the change bucketing the number of accidents, our model gave us 73% of correctly classified instances and 26% of incorrectly classified instances.
